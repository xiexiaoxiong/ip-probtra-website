# IP-Proposal

`IP-Proposal` 是一个给知识产权诉讼团队使用的 AI Agent Skill。它把“权利人 + 侵权链接/店铺/商品/内容”这类零散线索，整理成一份可给律师、客户或团队成员阅读的 `.docx` 诉讼方案报告。

它不是简单判断“侵不侵权”，而是先查权利、再看产品或内容相似性，最后在商标侵权、不正当竞争、专利侵权、著作权侵权之间选择最有利的诉讼路径。

## 它能做什么

输入权利人和侵权线索后，`IP-Proposal` 会尽量完成：

- 查询权利人侧的商标、专利、著作权、包装装潢或作品权属线索；
- 查询被诉方侧是否有自己的商标、专利、著作权或经营主体；
- 访问侵权链接，记录商品标题、店铺、SKU、价格、销量、评论、图片、详情页等线索；
- 对比原告产品和被告产品的包装、外观、页面素材或作品表达；
- 比较商标、不正当竞争、专利、著作权四条路径的胜诉稳定性和高判赔潜力；
- 设计被告结构、管辖法院、证据方案、赔偿模型和抗辩应对；
- 输出一份人类友好的 `.docx` 报告，而不是只在聊天框里堆检索过程。

## 最小输入

只要给出下面两个信息，就可以生成初版报告：

```text
用 $ip-proposal 评估并生成诉讼方案报告。
权利人：某品牌/某公司
侵权链接：https://example.com/item/123
```

如果侵权人、销量、权利证书、包装图片等信息暂时缺失，报告会标为 `待核验`，并列出下一步取证方法。

更完整的输入会显著提高报告质量：

- 侵权人、店铺名、生产商、销售商或关联公司；
- 原告产品图、被告产品图、包装图、页面截图；
- 商标证、专利证、著作权登记、授权链；
- 价格、销量、评价数、订单、发票、物流、客服记录；
- 目标法院、目标索赔额、既有公证或时间戳证据。

## 核心方法

`IP-Proposal` 的关键规则是：不要一上来就默认走商标侵权。

对产品类案件，它必须先完成：

1. 查权利人有哪些商标、专利、著作权和包装装潢基础；
2. 查被诉方有哪些商标、专利、著作权和经营主体；
3. 对比双方产品包装、外观、页面素材或表达；
4. 再决定主案由是商标、不正当竞争、专利还是著作权。

如果被诉商品使用的是自己的品牌，但整体包装明显模仿权利人的商品，主路径往往不是商标侵权，而是“有一定影响的商品包装装潢仿冒”不正当竞争。

## 在 Codex 中安装

最简单的方法不是手动安装，而是把下面这段话直接发给目标 agent，让它自己安装：

```text
请帮我安装 IP-Proposal skill。
GitHub 地址：https://github.com/xiexiaoxiong/ip-probtra-website/tree/main/skills/ip-proposal

请你自行完成安装：
1. 如果你支持 Codex/OpenAI 风格的 skill，请把该目录安装为 ip-proposal，并确保我可以用 $ip-proposal 调用；
2. 如果你不支持这种 skill 机制，请把 SKILL.md 和 references/ 作为你的提示词包/知识包，并配置 $ip-proposal 别名；
3. 安装后告诉我具体调用方式，并用一句话确认你会在完整评估时生成 DOCX 诉讼方案报告。
```

如果你是手动给 Codex 安装，也可以用下面的方式。

把整个 skill 目录复制到 Codex 的 skills 目录：

```bash
cp -R skills/ip-proposal ~/.codex/skills/
```

然后在 Codex 中这样调用：

```text
用 $ip-proposal 评估。
权利人：某品牌/某公司
侵权链接：https://example.com/item/123
```

Codex 会通过 `SKILL.md` 的 frontmatter 自动识别该 skill。正式 skill 名是 `ip-proposal`，人类展示名是 `IP-Proposal`。

## 在 OpenClaw 或其他 Agent 中安装

优先使用上面的“直接让 agent 安装”提示词。只有在目标 agent 不能联网、不能读 GitHub，或你要手动配置时，才需要看下面的步骤。

如果目标 agent 不支持 Codex 的 skill 自动发现机制，可以把本目录当作“提示词包”使用。

推荐安装方式：

1. 将 `skills/ip-proposal/` 复制到目标 agent 可读取的目录，例如 `~/.openclaw/skills/ip-proposal/`。
2. 在该 agent 的系统提示词、工具说明或自定义 skill 配置中加入：

```text
当用户调用 $ip-proposal，或要求评估知识产权侵权链接、生成诉讼方案、高判赔路径、证据方案、管辖和被告策略时：
1. 读取 ip-proposal/SKILL.md；
2. 若是完整案件评估，按 SKILL.md 的 Required References 依次读取 references/ 下的文件；
3. 对权利人和被诉方分别做商标、专利、著作权和包装/表达基础检索；
4. 对产品包装、外观或内容表达做相似性比对；
5. 在商标、不正当竞争、专利、著作权之间选择主路径和备选路径；
6. 生成一份人类友好的 .docx 诉讼方案报告。
```

3. 给 agent 配置必要能力：
   - 网页搜索和浏览器访问；
   - 图片识别或截图分析；
   - 本地文件读写；
   - `.docx` 生成能力；
   - 必要时允许用户登录平台页面，但不得绕过验证码、风控或权限控制。

也可以直接把下面这句设为别名：

```text
$ip-proposal = 读取 ip-proposal/SKILL.md 和 Required References，调查权利人及侵权目标，完成权利清单、包装/表达比对、案由选择、管辖被告、赔偿模型和证据方案，并输出 DOCX 诉讼方案报告。
```

## 输出格式

完整评估默认输出 `.docx` 报告。报告通常包括：

- 封面和执行摘要；
- 已核验事实、强推定事实、待核验事实；
- 权利人和被诉方权利清单；
- 产品包装、外观或作品表达对比；
- 商标 / 不正当竞争 / 专利 / 著作权路径评分；
- 推荐主路径和备选路径；
- 管辖法院、被告结构和平台策略；
- 赔偿模型、索赔区间和高判赔可行性；
- 证据清单、取证步骤和抗辩应对；
- 立即行动清单和来源记录。

最终聊天回复应尽量简短，只交付报告路径或下载链接。实质分析放在 Word 报告里。

## 注意事项

`IP-Proposal` 是诉讼策略和证据组织工具，不保证法院立案、胜诉或具体赔偿金额。正式起诉前，应由律师基于官方权利记录、公证证据、平台披露数据、实物证据和最新法律规则复核。
